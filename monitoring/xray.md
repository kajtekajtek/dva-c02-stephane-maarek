# AWS X-Ray

AWS X-Ray provides distributed tracing for applications, helping you debug latency issues and errors across microservices.

---

## Why X-Ray

Traditional "add more logs and redeploy" troubleshooting gets hard in distributed systems. X-Ray gives end-to-end request visibility:

- Service map and dependency visualization.
- Per-request latency and fault breakdown.
- Error and throttling analysis.
- Impact analysis (which users/routes/services are affected).

---

## Compatibility

X-Ray can be used with:

- AWS Lambda
- API Gateway
- Elastic Load Balancing
- ECS / Fargate
- Elastic Beanstalk
- EC2 and on-prem application servers

---

## Core tracing model

- **Trace**: complete request journey across services.
- **Segment**: trace portion generated by one service.
- **Subsegment**: finer-grained operations within a segment.

Metadata attached to traces:

- **Annotations**: indexed key-value pairs (searchable/filterable).
- **Metadata**: non-indexed context payload (not for indexed searches).

---

## Instrumentation

To use X-Ray effectively:

1. Instrument code with the X-Ray SDK (or compatible instrumentation layer).
2. Enable active tracing where applicable (for example Lambda).
3. Ensure daemon/agent and workload IAM permissions are correct.

SDKs capture downstream calls and HTTP activity with minimal code changes in many frameworks.

---

## Sampling

Sampling reduces cost and overhead by tracing only a subset of requests.

Default behavior includes:

- Reservoir (at least a baseline requests/sec).
- Percentage rate for additional requests.

You can define custom sampling rules without redeploying application code.

---

## APIs and permissions

### Write path (daemon/agent)

- `PutTraceSegments`
- `PutTelemetryRecords`

### Read path (analysis)

- `GetServiceGraph`
- `GetTraceSummaries`
- `BatchGetTraces`
- Additional sampling/statistics APIs

Common managed policy: `AWSXRayDaemonWriteAccess`.

---

## Deployment patterns

### Elastic Beanstalk

- Enable X-Ray daemon in platform configuration (for supported platforms).
- Ensure instance profile has required IAM permissions.
- Instrument app code with SDK.

### ECS and Fargate

Common patterns:

- Dedicated daemon service on ECS/EC2.
- Sidecar container per task/service (common on Fargate).

---

## Troubleshooting checklist

- X-Ray SDK/instrumentation included?
- Active tracing enabled (if needed)?
- Daemon/agent reachable and running?
- Correct IAM permissions for trace upload?
- Sampling rules too strict (dropping expected traces)?
