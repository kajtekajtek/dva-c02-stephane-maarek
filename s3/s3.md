# Amazon Simple Storage Service

## S3 Introduction

- **What is Amazon S3?**
  - Object storage service
  - Scalability, availability, security, performance
  - Store and protect data for many use cases
- Core AWS building block
  - Often described as effectively unlimited scale for typical workloads
- **How it works**
  - Data stored as **objects** inside **buckets**
  - Object = file + metadata
  - Each object has a **key** (identifier within the bucket)
- **Consistency**
  - Strong **read-after-write** consistency for `PUT` and `DELETE` on objects
  - Reads of ACLs, tags, metadata, S3 Select: strongly consistent
- Many AWS services integrate with S3
  - Analytics
  - ML
  - Logging [1]

### S3 Use Cases

Examples from AWS documentation:

- Data lakes
- Big data analytics
- Websites and static content
- Mobile applications
- Backup and restore
- Archive
- Enterprise applications
- IoT devices

Related product themes:

- Management features to optimize storage
- Organize and configure access
- Meet business and compliance requirements [1]

## Buckets

- **Container** for objects
  - Region-scoped
  - At creation: choose **bucket name** and **AWS Region**
  - After creation: **name** and **Region** cannot be changed
- **Types**
  - Each type targets different patterns
  - General purpose
  - Directory
  - Table
  - Vector
- **General purpose** (default for most cases)
  - Objects across storage classes (except S3 Express One Zone)
  - Redundancy across AZs
  - **Global namespace** per partition: bucket name unique across all accounts in that partition
- Optional **account regional namespace**
  - Names reserved for your account only
- **Directory buckets**
  - Low latency
  - Data residency
  - Hierarchical prefixes
  - S3 Express One Zone in AZ directory buckets
- **Table buckets**
  - Tabular data
  - Apache Iceberg
  - Analytics and ML
- **Vector buckets**
  - Vector embeddings
  - Similarity search
- Buckets **organize** the S3 namespace
- Buckets **identify** the account billed for storage and data transfer
- Buckets **aggregate** usage reporting [1]

- **Private by default**
  - Access only when you grant it
  - Policies
  - IAM
  - ACLs
  - Access points [3]

- **Block Public Access**
  - Four settings
  - On by default for new general purpose buckets
  - Can override policies and ACLs to cap public exposure [2]

## Objects

- **Fundamental** unit stored in S3
  - **Data**
  - **Metadata**: name-value pairs
    - System default
    - HTTP-style fields (e.g. `Content-Type`)
    - Optional user-defined metadata
- **Identity** in a bucket
  - **Key**
  - **Version ID** (when Versioning is on)
- **Key**: name used to retrieve the object
  - **Prefix** + **object name**
  - `s3://my-bucket/wielki/mistrz/ulryk.txt`
  - **Prefix**: `wielki/mistrz/`
  - **Object name**: `ulryk.txt`
- **Value**: object bytes
  - Size from zero up to **50 TB**
  - With a single PUT operation, you can upload a single object **up to 5 GB in size**
  - Bigger files can get uploaded using multipart upload operation
- **Subresources** (e.g. ACL, legal hold): attach to bucket or object; not standalone
- **Access control**
  - Resource-based: ACL, bucket policy
  - Identity-based: IAM
  - **Private by default**
  - Presigned URLs for time-limited sharing
- **Tags**: optional categorization
  - Access control
  - Cost allocation [4]
